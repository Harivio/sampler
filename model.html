Attribute, Object & Function Model

Objects

AudioContext

The AudioContext object represents the environment for managing and playing all audio within the project. It's the starting point for creating and processing audio.

Key Attributes:

state

Represents the current state of the AudioContext (e.g., running, suspended, closed).

Key Functions:

createGain()

Creates a GainNode used to control the volume of audio.

createBiquadFilter()

Creates a BiquadFilterNode for applying specific frequency filters.

createConvolver()

Creates a ConvolverNode for reverb effects.

decodeAudioData(arrayBuffer)

Decodes audio file data into a format that can be used by the AudioContext.

resume()

Resumes the audio context if it is suspended.

GainNode

The GainNode is used to control the volume of an audio signal.

Key Attributes:

gain

The gain (volume) of the node, typically a value between 0 (mute) and 1 (full volume).

Key Functions:

connect(destination)

Connects the gain node to another audio node, allowing the audio to flow through.

disconnect()

Disconnects the gain node from its destination, stopping the audio flow.

ConvolverNode

A ConvolverNode is used to apply reverb effects to an audio signal.

Key Attributes:

buffer

The audio buffer used for the reverb effect.

Key Functions:

connect(destination)

Connects the convolver node to another audio node.

disconnect()

Disconnects the convolver node from its destination.

AudioBuffer

An AudioBuffer represents a short audio asset residing in memory, used by the AudioContext.

Key Attributes:

length

The length of the buffer in sample frames.

duration

The duration of the buffer in seconds.

sampleRate

The sample rate of the buffer in Hz.

Key Functions:

Mainly data storage.

AudioBufferSourceNode

An AudioBufferSourceNode is used to play an AudioBuffer in the audio context.

Key Attributes:

buffer

The audio buffer that contains the sound to be played.

Key Functions:

connect(destination)

Connects the source node to another audio node.

start()

Begins playback of the audio buffer.

HTML Elements (Grids, Buttons, Sliders)

These elements provide the user interface for interacting with the sampler.

Key Attributes:

dataset

Custom data attributes used to store sound URLs and other metadata.

value

The current value of input elements (e.g., sliders).

textContent

The text content displayed within an HTML element.

classList

The list of CSS classes associated with the element, which can be manipulated to change styles dynamically.

Key Functions:

addEventListener(event, handler)

Attaches an event handler to the element to listen for specific events like clicks or input changes.

querySelector(selector)

Returns the first matching element within the element.

click()

Simulates a mouse click on the element, useful for triggering actions programmatically.

classList.add(className)

Adds a class to the element's class list.

classList.remove(className)

Removes a class from the element's class list.

Functions

Initialization Functions

initializeAudioContext()

This function initializes the audio context, making it ready to process and play audio in response to user interaction.

loadReverbImpulse(url)

Loads a reverb impulse response from the specified URL, decoding it for use in the ConvolverNode.

Audio Functions

loadAudio(url)

Fetches and decodes audio data from a given URL, returning an AudioBuffer that can be played in the audio context.

playAudio(buffer, individualGain)

Creates an audio buffer source and plays it through the gain and filter nodes, producing sound output.

Event Handlers

handleReverbChange(event)

Adjusts the reverb gain based on user input from the reverb slider.

handleVolumeChange(event)

Adjusts the master gain based on user input from the master volume slider.

handleEQChange(event, filterNode)

Adjusts the equalizer filter gains based on user input, corresponding to the low, mid, and high EQ sliders.

handleGridClick(grid)

Plays the sound associated with the grid element and provides visual feedback for user interaction.

provideVisualFeedback(grid)

Provides visual feedback when a grid is clicked by adding an "active" class to the grid element and then removing it after a short delay.

handleFileUpload(event, grid)

Loads and decodes an audio file uploaded by the user, associating it with the corresponding grid element.

handleGridVolumeChange(event, individualGain)

Adjusts the gain of the individual sound based on user input from the grid volume slider.
